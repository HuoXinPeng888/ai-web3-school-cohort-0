### 区块链基础概念

**区块链** 是一种分布式账本技术，核心特点：
- 去中心化：没有单一控制者
- 不可篡改：数据一旦上链难以修改
- 透明可查：所有人都可以查看链上数据
- 共识机制：网络参与者共同验证交易

**以太坊（Ethereum）**
- 第二代区块链，支持智能合约
- 原生代币：ETH（以太币）
- 虚拟机：EVM（以太坊虚拟机）
- 智能合约语言：Solidity、Vyper

**关键术语**

| 术语 | 说明 |
|------|------|
| 钱包（Wallet） | 管理账户和密钥的工具，如 MetaMask |
| 地址（Address） | 0x 开头的 42 位字符串，账户标识 |
| 私钥（Private Key） | 控制账户的密钥，必须保密 |
| 助记词（Seed Phrase） | 12/24个单词，用于恢复钱包 |
| 智能合约（Smart Contract） | 部署在链上的自动执行程序 |
| Gas | 执行交易需要的手续费 |
| Gas Fee | 用 ETH 支付的交易费用 |

### 智能合约入门

**什么是智能合约？**
- 部署在区块链上的一段代码
- 当满足特定条件时自动执行
- 代码即法律（Code is Law）

**第一个智能合约：简单存储**

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

contract SimpleStorage {
    uint256 private storedNumber;

    function set(uint256 _number) public {
        storedNumber = _number;
    }

    function get() public view returns (uint256) {
        return storedNumber;
    }
}
```

**代码解析**
- `SPDX-License-Identifier`: 许可证声明
- `pragma solidity`: 指定 Solidity 版本
- `contract`: 合约关键字，类似 class
- `uint256`: 无符号256位整数
- `public`: 公开可见，外部可调用
- `view`: 只读函数，不修改状态
- `returns`: 指定返回值类型

### Remix IDE 使用指南

**打开方式**
访问：https://remix.ethereum.org/

**左侧面板**
1. 文件浏览器 - 创建和管理合约文件
2. 编译器 - 编译 Solidity 代码
3. 部署和运行 - 部署合约、调用函数

**部署环境选择**
- **Remix VM (London)**: 本地虚拟机，10个测试账户，无 gas 费 ✅ 推荐新手
- Injected Provider - MetaMask: 连接 MetaMask 钱包
- Hardhat Provider: 连接本地 Hardhat 节点

**操作步骤**
1. 在文件浏览器中创建 `.sol` 文件
2. 编写智能合约代码
3. 切换到编译器面板，点击 "Compile"
4. 切换到部署面板，选择环境
5. 点击 "Deploy" 部署合约
6. 在下方 "Deployed Contracts" 中调用函数
